Pricing

Thanks to this is a high performance implementation us servers are able to manage thousands of requests per second resulting in very competitive prices.
The prices below are expressed by shard, each shard is able to handle up to the specified request per second, and store up to the specified number of elements.
The elements stored in a server are the number of classifications stored on it, for instance, if 20 different users has classify an average of 5 elements, we will have 100 elements stored in total. Pitia uses a LRU policy in order to store in memory the newest classifications removing the old classifications when the limit of the number of elements are reached, on this way the system is able to adapt the recommendations to the last trends in your application.

All the prices are expressed by shard. Remember that the first shard of a group is always composed by two shards but you will be charged just by one. Please have in consideration that one of the shards can be off-line, in order to provide the best user experience to your clients, please try to reserve enought shards to cover all the necessities when one of the shards is down.
